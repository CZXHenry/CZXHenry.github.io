如何利用Hexo搭建博客
2018-10-14

之前使用过Jeklly+Git Page搭建个人博客，但是不得不说Jeklly是真的太难用了，有时连一些很基本的markdown语法都不支持，更加不用说公式之类的。后来听说Hexo不错，这两天捣鼓了一下，不得不说各种Hexo主题真的是高大上，现在也弄得七七八八了，所以大概记录一下搭建过程吧。

安装Hexo

1. 安装 Node.js

因为这个我之前已经装好了，所以具体装法就要靠大家摸索了

2. 安装 Git

这个到Git官网上看看教程装下应该可以了，注意下将密钥添加到GitHub之后可能会出现的 Permission Denied 问题，我记得好像是缺少某个文件造成的，新建一个就行，具体还是百度吧2333

3. 安装Hexo

  • 确保安装了 Node.js 和 Git
  • 打开命令行（不需要管理员权限），输入命令 npm install -g hexo-cli
  • 找到/新建一个文件夹作为hexo的安装文件夹
  • 转到hexo安装文件夹，打开命令行，输入 hexo i blog
  • 等待安装完成之后，转到blog文件夹下
  • 输入 hexo g 和 hexo s 命令，分别生成静态页面和开启本地服务器
  • 打开命令行中提示的本地访问地址（应该都是localhost:4000的了），就可以看到样例博客了

个性化

因为Hexo自带的主题感觉不是很好看（单纯想折腾），所以想要换个主题。我个人认为先在本地把自己想要的博客样式弄好之后再传到GitHub上面可能更好，不用每修改一次都需要push到GitHub上。

1. 更换主题

  • 首先打开 Hexo官方的主题页面，上面有好多不同的主题任你挑选。
  • 挑选到合适的主题之后（以下以我安装的 Archer 主题为例），点击进入该主题的示例网站
  • 进入示例网站之后，会发现主题作者把这个主题的安装方法已经放在这个示例网站里面的了（应该都会放的，我看的几个都有放）
  • 然后按照作者所写教程一步步安装即可,进一步的个性化就看自己了，这个主题基本能满足所有需求了吧

部署到GitHub上

1. 设置Git Page

这个比较简单，百度一下就行了

2. 将本地博客部署到GitHub上

  • 输入命令 npm install hexo-deployer-git --save 安装一个帮助我们部署和发布博客的插件
  • 等插件安装好之后，使用命令 hexo d 将博客部署到GitHub上
  • 访问Git Page，看看是不是已经装上去了

发布文章

部署好了博客，发文章肯定是必须的啦

1. 安装插件并发布

  • 输入命令 ‘hexo new “文件名”‘，就会自动在 blog/source/_posts 文件夹下。手动添加新的文章也可以
  • 编辑文章
  • 文章编辑完之后再次 ‘hexo g’ 和 ‘hexo d’
  • 对于本地服务器来说，每次更新博客外观或者更新文章，更改都是刷新下网页即时可见的；但是要使在Git Page上面生效，必须先输入 hexo g 命令再使用 hexo d 命令
